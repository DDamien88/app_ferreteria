# Proyecto App movil de gestión de ferretería ( Administradores).


# Introducción

**Este proyecto consiste en el desarrollo de una aplicación móvil nativa para Android, destinada a dueños y/o encargados de una ferretería. Su objetivo es optimizar las tareas de control de stock, administración de productos y visualización de reportes de ventas, sin realizar operaciones de venta desde la app móvil (las ventas se gestionan en la versión web).**

## Justificación

**La gestión de inventario en una ferretería requiere actualizaciones constantes. Una app móvil permite realizar estas tareas desde cualquier lugar del local, sin depender de una computadora. Esto mejora la eficiencia, el control de stock y reduce errores de registro.**

## Objetivo general del proyecto

**Desarrollar una aplicación móvil Android que permite a dueños/encargados de una ferretería administrar productos, controlar stock y visualizar reportes de ventas, conectándose a un backend centralizado.**

## Objetivos específicos del proyecto

-   Permitir el alta, edición y baja lógica de productos.  
      
    
-   Ajustar stock mediante registros de entrada y salida.  
      
    
-   Consultar reportes de ventas (solo lectura).  
      
    
-   Recibir notificaciones sobre productos con stock bajo.  
- Gestionar los datos personales del usuario logueado.

## Objetivo general del sistema

Gestionar la información interna de la ferretería mediante una aplicación móvil conectada a un servidor web (API REST), cubriendo tareas administrativas de inventario y reportes.

## Límite

-   La app no realiza ventas.  
      
    
-   No genera facturas ni boletas.  
      
    
-   No está orientada al cliente final.  
      
    
-   No gestiona múltiples sucursales (en esta versión).
